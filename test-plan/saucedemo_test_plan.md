# SauceDemo Test Plan

## 1. Objective
Validate core functionality, usability, and reliability of SauceDemo web app. Focus testing on user flows to identify bugs and defects, ensuring a reliable user experience.

## 2. Application Being Tested
- Name: SauceDemo
- URL: https://www.saucedemo.com
- Type: Web application
- Function: E-commerce demo website for testing

## 3. Scope

### In Scope
- User authentication - login/logout
- Product sorting and browsing
- Cart functionality
- User Checkout
- UI validation
- Error handling/reporting

### Out of Scope
- Perfomance and load
- Security
- Backend and database
- Mobile responsiveness

## 4. Test Approach
- Manual functional testing
- Exploratory testing on edge-casing
- Browser DevTools testing in Chrome
- Validation of positive and negative scenarios

## 5. Test Environment
- Browser: Google Chrom (most recent version)
- OS: Desktop environment
- Network: Standard consumer internet connection

## 6. Test Data
- Valid/invlaid login credentials (SauceDemo provided)
- Sample products in the application

## 7. Entry Criteria
- Application is accessible
- Test environment is available
- Test data is known

## 8. Exit Criteria
- All planned test cases executed
- Critical/severe defects/bugs documented
- Test results recorded

## 9. Risks and Assumptions
- Demo site may intentionally contain defects/bugs
- Limited test environment might miss non-Chrome issues

## 10. Deliverables
- Test plan
- Test cases
- Bug reports
- Exploratory testing notes
